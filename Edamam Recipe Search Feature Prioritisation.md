# Feature Prioritisation for Edamam Recipe Search Program

---

**Must have** 

- [x] Ability to search for recipes by ingredient  
- [x] Display basic information about each search result such as the recipe name and url  
- [x] Error checking and handling for all API calls. This includes handling scenarios where the API is unavailable, the requested ingredient doesn’t exist, or the API response is not as expected.  
- [x] Informative error messages for the user, so they understand if something went wrong and why.

---

**Should have**

- [x] Information output is relevant, user-friendly and well-formatted  
- [x] Requests for input are clear and specify the type of format  
- [x] Error handling and feedback are implemented in the case of incorrect or invalid input  
- [x] If no results are found, helpful feedback is given

---

**Could have**

* Ability to save the search results to a file  
* Order the results by weight or another piece of data  
* Ask the user additional questions to decide which recipe they should choose  
* Cross-reference the ingredient against the Edamam nutrition analysis API  
* Ability to list more than 10 recipe results  
* Ability to display next and previous pages

---

**Won’t have**

* Ability to display recipe images  
* Ability to list more than 10 recipe results on a page  
* Graphical interface
